## 1. 정보보안 개요

### 🔹 정보보안의 정의
- **정보보안(Information Security)**: 정보의 기밀성, 무결성, 가용성을 유지하여 안전하게 보호하는 것.
- **정보보안의 3대 요소 (CIA)**
  - **기밀성(Confidentiality)**: 허가된 사용자만 정보에 접근 가능
  - **무결성(Integrity)**: 정보가 위·변조되지 않고 유지됨
  - **가용성(Availability)**: 필요할 때 정보에 접근 가능

---

## 2. 암호학 (Cryptography)

### 🔹 암호 알고리즘 분류
1. **대칭키 암호화 (Symmetric Encryption)**
   - 암·복호화에 같은 키를 사용
   - 속도가 빠르지만, 키 분배가 어려움
   - 대표 알고리즘: **DES, AES, SEED**

2. **비대칭키 암호화 (Asymmetric Encryption)**
   - 공개키(Public Key)와 개인키(Private Key)를 사용
   - 속도가 느리지만, 키 분배가 용이
   - 대표 알고리즘: **RSA, ECC, ElGamal**

3. **해시 함수 (Hash Function)**
   - 입력값을 고정된 길이의 해시값으로 변환
   - 대표 알고리즘: **MD5, SHA-256**

---

## 3. 네트워크 보안

### 🔹 네트워크 공격 유형
1. **DoS/DDoS 공격**
   - 서비스 거부 공격으로 시스템 리소스를 과부하 상태로 만들어 서비스 방해
   - 대표적인 공격 기법: **SYN Flooding, UDP Flooding, HTTP GET Flooding**

2. **스니핑 (Sniffing)**
   - 네트워크 트래픽을 가로채서 정보를 탈취하는 공격
   - 대표 도구: **Wireshark, tcpdump**

3. **스푸핑 (Spoofing)**
   - 신뢰할 수 있는 시스템으로 위장하여 불법적인 접근 시도
   - 종류: **IP Spoofing, ARP Spoofing, DNS Spoofing**

4. **MITM (Man-In-The-Middle) 공격**
   - 공격자가 통신 중간에서 데이터를 가로채거나 조작하는 공격
   - 방어 기법: **SSL/TLS 암호화, VPN 사용**

---

## 4. 시스템 보안

### 🔹 운영체제(OS) 보안 기법
- **접근 제어 (Access Control)**: 사용자 권한을 제한하여 보안 강화
- **로그 모니터링**: 시스템 로그 분석을 통한 침입 탐지
- **패치 관리**: 최신 보안 패치 적용을 통한 취약점 제거

---

## 5. 웹 보안

### 🔹 주요 웹 보안 취약점 (OWASP Top 10)
1. **SQL Injection**: SQL 문을 조작하여 DB 데이터를 탈취
2. **XSS (Cross-Site Scripting)**: 악성 스크립트를 삽입하여 사용자 세션 탈취
3. **CSRF (Cross-Site Request Forgery)**: 사용자의 세션을 가로채어 원치 않는 요청을 수행하게 함
4. **파일 업로드 취약점**: 악성 코드가 포함된 파일 업로드를 통한 시스템 공격

---

## 6. 보안 관리 및 법규

### 🔹 정보보호 관련 법규
- **개인정보 보호법**: 개인정보 수집, 이용, 보호에 대한 법률
- **정보통신망법**: 정보통신망 이용촉진 및 정보보호 등에 관한 법률
- **전자서명법**: 전자서명의 법적 효력 및 인증기관 운영에 관한 법률

2) find / -user root -type f \( -perm -4000 -o -perm -2000 \) |xargs ls -al

 1) 특정 파일에 설정된 특수 비트(SetUID, SetGID) 를 제거함.

 - 특수비트가 설정된 파일을 실행하는 경우 파일의 소유주(SetUID 설정시), 또는 소유그룹(SetGID 설정시) 권한으로 실행됨. 특히 root 가 소유주 또는 소유 그룹인 파일에 특수비트가 설정되면, 일반 사용자 계정으로 실행하더라도 root 권한으로 실행되어 악의적인 행위가 가능하기 때문임.
